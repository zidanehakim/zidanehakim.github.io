How it breaks down

1

Scan trigger on scroll-into-view

Framer Motion's useInView fires the scanner sequence once when the section enters the viewport. Replays on re-entry.

Framer Motion
2

Purple beam sweeps across name

CSS animation clips a colored version of "Yazidane Hakim" from left→right. The beam is a glowing 2px line in sync.

CSS clip-path
3

Measurement pills stagger in

w / h / baseline / leading pills fade+slide up as the beam passes, staggered 300ms apart. Values are hardcoded (no live SDK call needed).

stagger animation
4

Constellation radiates from name

Canvas draws animated node-graph — nodes are layout properties, edges pulse. Nodes appear progressively after scan completes.

Canvas 2D
5

Lottie sparkle loops in corner

Lightweight sparkle/glow Lottie (from lottiefiles.com, ~30kb) loaded via dynamic() — zero impact on initial load.

Lottie lazy-loaded

Copy + context

Below the animation, a short paragraph grounds it for the employer:

// built with PRETEXT SDK

PRETEXT by Cheng Lou enables real-time multiline text measurement — tracking width, height, baseline, and leading as layout changes. I use it to drive pixel-perfect responsive typography throughout this site.

Placement: Sits between the About intro and the Timeline section. Full-width dark zone (~60vh) so it reads as a dedicated showcase, not a footnote.

ASCII corners frame the section — connects the terminal aesthetic to the rest of the portfolio without feeling out of place.

PRETEXT npm package

Will import PRETEXT SDK and use measure() to compute real values from the actual rendered text — so the numbers in the pills are live, not fake.
