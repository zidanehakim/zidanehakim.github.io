Page Structure

How should the site be organized? This affects routing, scroll behavior, and the overall feel.

A

Multi-page (current approach, modernized)

Keep 4 separate routes: / /about /portfolio /contact. Each page is a full-screen view with the floating pill navbar. Fast page transitions via Framer Motion. SSG-friendly for GitHub Pages.

/ → Hero + typewriter
/about → Bio + Timeline + Skills
/portfolio → Projects carousel
/contact → Contact info
+ PRETEXT section added as a full page /pretext or embedded in About
B

Single-page scroll (modern portfolio standard)

One long scrollable page with sections: Hero → About → Skills → PRETEXT Showcase → Projects → Contact. Floating nav updates active state on scroll. Smooth scroll-snapping between sections.

↓ #hero
↓ #about + #timeline
↓ #pretext-showcase ← ✨ standout section
↓ #projects
↓ #contact
Single route / — no transition delays between sections

Router decision (not visual)

Recommendation: Next.js App Router for both options. TanStack Router adds no real value for a static portfolio on GitHub Pages. TanStack Query will still be used for any async data (GitHub API for projects, etc.).
